# EFL Match Prep Tool

A broadcast match preparation tool for EFL and World Cup football coverage. Pick a fixture, get data, plan your production.

## What it does

Select a Championship, League One, League Two, or World Cup fixture by date. The tool loads pre-built static data files (generated by the build-time scrapers) to populate:

- **Banner** — team crests, team colours, competition name, venue, date
- **Manager cards** — name, appointment date
- **Form** — last 5 results with scorers, most recent match detail
- **Key players** — 5 editable slots per team, autocomplete from squad list, persisted per match
- **Scout notes** — freetext tactical notes per team, persisted per match
- **Injuries / availability** — loaded from `data/injuries.json`; freetext override persisted per match
- **Kit colours** — loaded from `data/kits.json`; falls back to colour blocks
- **Expected formations** — landscape pitch view, populated from the Confirmed XI tab
- **Head to head** — win/draw/loss record + most recent meetings with scorers
- **League table** — full standings with both teams highlighted, promotion/playoff/relegation zones
- **Squad lists** — sortable by number or surname

**Production tab**
- Ground layout diagram — place cameras, dugouts, away fans, tunnel by clicking; drag to reposition; saved per venue
- Ground notes — freetext, saved per venue, reloads automatically for future matches at the same ground
- Camera operators — name (with autocomplete from history), star rating, match notes; rating history tracked across matches

**Confirmed XI tab**
- Enter formation and players for each team
- Autocomplete from squad data
- "Fill from Squad" — auto-distributes squad by position into the selected formation
- "Import API Lineup" — pulls confirmed XI from the API once published (~1hr pre-kick-off)
- Formation view updates live in the Match Data tab

**Print / PDF**
- Browser print → A4 landscape, single page — managers, form, scout notes, formations, squad lists

## Data pipeline

Data is built ahead of the match using the Sofascore scraper (no API key required). Run it the morning before the game.

```bash
# List fixtures on a date
node scraper-sofascore.js --comp EL1 --date 2026-08-10

# Fetch all data for a specific match
node scraper-sofascore.js --match 14060002
```

This writes five files to `data/` — the app loads them at startup:

| File | Contents |
|---|---|
| `data/teams.json` | Squad, manager, crest URL, team colours per team |
| `data/tables.json` | League standings per competition |
| `data/h2h.json` | Head-to-head aggregate + match history per fixture |
| `data/form.json` | Last 10 finished matches per team |
| `data/matches.json` | Fixture index (match ID → basic match object) |
| `data/kits.json` | Kit image URLs per team (from `scraper-kits.js`) |
| `data/injuries.json` | Injury/availability per team (from `scraper-injuries.js`) |

### Competitions supported

| Competition | Scraper code |
|---|---|
| Sky Bet Championship | `ELC` |
| Sky Bet League One | `EL1` |
| Sky Bet League Two | `EL2` |
| FIFA World Cup 2026 | `WC` |

### Fallback scraper

`scraper-data.js` uses the football-data.org API as a fallback. Requires `FOOTBALL_DATA_KEY` in a `.env` file. Free tier covers Championship + World Cup only; League One/Two requires Tier 2 (€49/month).

```bash
node scraper-data.js --comp WC --date 2026-06-15
node scraper-data.js --match 521234
```

## Files

| File | Purpose |
|---|---|
| `index.html` | Markup — header, tabs, match selector, ground layout SVG |
| `style.css` | All styles including print layout |
| `app.js` | All JavaScript — data loading, rendering, localStorage |
| `scraper-sofascore.js` | Primary build-time scraper (Sofascore, no key required) |
| `scraper-data.js` | Fallback scraper (football-data.org, requires API key) |
| `scraper-kits.js` | Kit image scraper for EFL clubs |
| `scraper-kits-wc.js` | Kit image scraper for WC nations |
| `scraper-injuries.js` | Injury/availability scraper (Soccerway) |
| `worker.js` | Cloudflare Worker proxy for live API fallback |

## localStorage keys

| Key | Contents |
|---|---|
| `groundLayout_{venueSlug}` | Camera/marker positions per venue |
| `groundNotes_{venueSlug}` | Ground notes per venue |
| `xi_home_{matchId}` / `xi_away_{matchId}` | Confirmed XI per match |
| `kp_home_{matchId}` / `kp_away_{matchId}` | Key player entries per match |
| `scout_home_{matchId}` / `scout_away_{matchId}` | Scout notes per match |
| `inj_home_{matchId}` / `inj_away_{matchId}` | Injury notes per match |
| `camop_{0-3}_{matchId}` | Camera operator name, rating, notes per match |
| `camOperatorsDB` | Global list of all operator names (autocomplete) |
| `camopHist_{name}` | Running rating average per operator |

## Design

- **Fonts:** Bebas Neue (headings), Barlow Condensed (labels/UI), DM Sans (body), DM Mono (stats)
- **Colours:** Clean, data-forward — white surfaces, warm off-white background, team colours as accents
- **Formation layout:** Landscape, GK left→FWD right for home; mirrored for away so both teams face each other
